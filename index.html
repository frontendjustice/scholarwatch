The Pipeline

Eight steps from raw RSS to actionable scholarship alerts.

1
📡

RSS Fetching

Reads active feed URLs from the database, parses each with feedparser, cleans HTML from descriptions, and returns a flat list of entries.

Input: Feed URLs
Output: Raw entries
▼
2
🔄

Deduplication

Removes entries with identical link + title combinations within the same run. Prevents the same opportunity from being processed twice.

Filter: link + title hash
▼
3
🎯

Smart Filtering

Each entry is scored on 4 criteria. Minimum score to accept: 3/4.

💰 Fully Funded
🌍 Open to LMIC / Everyone
🏥 Health-Related / All Courses
📅 Valid Deadline
Edge cases (2/4): Vague-but-promising entries are scraped and re-evaluated by AI with full page content.
▼
4
🕷️

Concurrent Scraping

All accepted entry URLs are scraped concurrently before any AI summary calls. This batches network I/O for efficiency.

Async: up to 10 concurrent
▼
5
🤖

Deep Summary

4/4 matches get a full AI-generated 6-sentence summary from scraped page content. 3/4 matches get the cleaned RSS description with deadline extracted — no AI call to save tokens.

AI: only for 4/4 scores
Token-efficient
▼
6
📅

Deadline Extraction

Regex scans raw page text for date patterns before the AI prompt runs. Found date is passed to AI as a confirmed hint, improving accuracy.

Output: ISO date or "Not found"
▼
7
💾

Save to Database

Screened results are persisted to SQLite with full metadata: title, link, score, summary, deadline, source feed, and timestamp.

Storage: SQLite
▼
8
⏰

Daily Run

The entire pipeline runs on a scheduled trigger. New results are surfaced in the dashboard; previously-seen entries are skipped.

Schedule: daily
